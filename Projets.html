// SAÉ 1.05 — Développement
SAÉ 1.05 — DÉVELOPPEMENT
Outil d'Analyse & Nettoyage de Disque
Python PowerShell PyQt5 JSON

Solution logicielle modulaire développée dans le cadre de la SAÉ 1.05. L'objectif était de concevoir un outil permettant d'identifier les fichiers volumineux sur un serveur et de les supprimer via une interface graphique sécurisée. Le projet repose sur trois composants distincts qui communiquent entre eux. L'interface intègre un graphique en camembert pour visualiser la répartition des tailles détectées, ainsi qu'une vue liste affichant les chemins complets et la taille précise de chaque fichier identifié.

Architecture du projet
  • sae.ps1 — Orchestrateur PowerShell : préparation de l'environnement, vérification des dépendances, montage volumes réseau (New-PSDrive)
  • analyse.py — Moteur Python : parcours récursif (os.walk), tri par taille, export JSON
  • main_gui.py — Interface PyQt5 : affichage graphique (camembert), tableau des fichiers, suppression sécurisée
Problèmes résolus
  • Chemins UNC — montage automatisé via PowerShell pour les dossiers réseau
  • Précision numérique — division flottante + round() pour les fichiers < 1 Mo
  • Communication inter-processus PowerShell ↔ Python
  • Sécurisation de la suppression (confirmation utilisateur)
Objectifs pédagogiques
  • AC11.02 — Algorithmes complexes (tri, récursivité), structures JSON
  • Scripting système avancé et interactions inter-processus
  • Conception IHM orientée utilisateur avec Qt
Compétences acquises
  • Programmation Python avancée — os.walk, json, sorted()
  • Scripting PowerShell — New-PSDrive, gestion d'erreurs
  • Développement d'interfaces graphiques PyQt5
  • Architecture modulaire et séparation des responsabilités

SAÉ 1.02 — RÉSEAUX
Architecture Réseau d'Entreprise
EVE-NG Cisco IOS VLAN / VTP DHCP Linux

Conception et déploiement complet d'un réseau local d'entreprise sous EVE-NG avec matériel Cisco simulé. L'objectif était de mettre en place une architecture réseau structurée, segmentée et fonctionnelle, intégrant VLAN, VTP, DHCP Linux et routage inter-VLAN. Les captures ci-dessous illustrent la topologie simulée sous EVE-NG, la configuration du service DHCP Linux pour l'attribution dynamique des adresses IP par VLAN, et la segmentation réseau au niveau du switch.

Implémentation technique
  • Segmentation réseau — VLANs Direction, Employés, Wi-Fi Invités
  • Propagation automatique des VLANs via protocole VTP
  • Serveur DHCP Linux Debian — attribution dynamique par VLAN (DHCP relay)
  • Routage inter-VLAN — Router-on-a-Stick ou SVI selon topologie
  • Vérification de connectivité complète (ping, traceroute)
Environnement utilisé
  • EVE-NG — virtualisation réseau complète
  • Routeurs & switches Cisco IOS
  • Serveur Linux Debian 12 pour le DHCP
  • VPC pour simulation des postes clients
Objectifs pédagogiques
  • AC11.03 — Configurer les fonctions de base du réseau local
  • Maîtrise de la CLI Cisco IOS & commutation
  • Virtualisation avancée avec EVE-NG
Compétences acquises
  • Configuration CLI Cisco — VLAN, VTP, trunk, access
  • Administration DHCP Linux en multi-VLAN
  • Diagnostic réseau — show vlan, show ip route, debug
  • Analyse et résolution de problèmes de connectivité

PROJET PERSONNEL — AUTOMATISATION
Agent IA / Automatisation n8n
n8n RSS IA générative Automatisation

Workflow d'automatisation construit dans n8n pour la veille technologique automatisée. Le pipeline récupère des articles via un flux RSS, applique un filtre limitant le nombre d'entrées traitées, puis transmet les articles sélectionnés à un modèle de langage qui génère des résumés synthétiques. Ce projet illustre une approche concrète de l'automatisation : traitement structuré de l'information, intégration d'outils et logique de flux.

Architecture du workflow
  • Déclencheur planifié — exécution automatique du workflow à intervalle défini
  • Nœud RSS — récupération des articles depuis un flux RSS
  • Nœud Limit — contrôle du volume traité par exécution
  • Nœud modèle — génération de résumés automatiques via un LLM
Logique du flux
  • Chaînage séquentiel des nœuds — chaque sortie alimente l'entrée suivante
  • Limitation du débit pour éviter un traitement excessif
  • Prompt structuré pour obtenir des résumés exploitables
  • Facilement extensible — ajout de nœuds d'alerte, de stockage ou de notification
Compétences mobilisées
  • Construction et débogage d'un workflow n8n
  • Intégration d'une source de données externe (flux RSS)
  • Utilisation pratique d'un modèle de langage dans un pipeline
  • Logique de traitement de l'information par étapes
Apport concret
  • Automatisation d'une tâche de veille répétitive
  • Réduction du temps de traitement manuel des flux d'information
  • Approche structurée : déclencheur → collecte → filtrage → traitement
  • Démonstration d'une intégration d'outils no-code / low-code efficace
